## Introduction

This blog now supports interactive UI components that make learning more engaging and hands-on. Below are demonstrations of all the interactive components you can use in your blog posts.

## Code Playground

The Code Playground lets readers write and execute JavaScript code directly in the browser:

<CodePlayground 
  title="JavaScript Playground"
  initialCode={`// Try modifying this code and clicking Run!
const fibonacci = (n) => {
  if (n <= 1) return n;
  return fibonacci(n - 1) + fibonacci(n - 2);
};

// Calculate first 10 Fibonacci numbers
for (let i = 0; i < 10; i++) {
  console.log(\`F(\${i}) = \${fibonacci(i)}\`);
}`}
/>

You can also create simpler examples:

<CodePlayground 
  title="Array Methods Demo"
  initialCode={`const numbers = [1, 2, 3, 4, 5];

// Map example
const doubled = numbers.map(x => x * 2);
console.log('Doubled:', doubled);

// Filter example
const evens = numbers.filter(x => x % 2 === 0);
console.log('Evens:', evens);

// Reduce example
const sum = numbers.reduce((acc, x) => acc + x, 0);
console.log('Sum:', sum);`}
/>

## Tabbed Content

Organize related content with tabs:

<Tabs 
  tabs={[
    {
      label: "Python",
      content: (
        <div>
          <h4>Python Implementation</h4>
          <pre className="language-python">
            <code>{`import numpy as np

def sigmoid(x):
    return 1 / (1 + np.exp(-x))

def relu(x):
    return np.maximum(0, x)

# Example usage
x = np.array([-2, -1, 0, 1, 2])
print(f"Sigmoid: {sigmoid(x)}")
print(f"ReLU: {relu(x)}")`}</code>
          </pre>
        </div>
      )
    },
    {
      label: "JavaScript",
      content: (
        <div>
          <h4>JavaScript Implementation</h4>

  return 1 / (1 + Math.exp(-x));

function relu(x) {
  return Math.max(0, x);
}

// Example usage
const values = [-2, -1, 0, 1, 2];
console.log('Sigmoid:', values.map(sigmoid));
console.log('ReLU:', values.map(relu));`}</code>

        </div>
      )
    },
    {
      label: "Math",
      content: (
        <div>
          <h4>Mathematical Definitions</h4>

          <p>σ(x) = 1 / (1 + e^(-x))</p>
          <br />
          <p><strong>ReLU Function:</strong></p>
          <p>ReLU(x) = max(0, x)</p>
        </div>
      )
    }
  ]}

## Combining Components

You can combine multiple components to create rich, interactive learning experiences:

### Neural Network Training Visualization

<Tabs 
  tabs={[
    {
      label: "Concept",
      content: (
        <div>
          <h4>Understanding Gradient Descent</h4>
          <p>Gradient descent is an optimization algorithm used to minimize the loss function in neural networks. It iteratively adjusts parameters in the opposite direction of the gradient.</p>
        </div>
      )
    },
    {
      label: "Visualization",
      content: (
        <div>
          <InteractiveChart 
            title="Loss Over Epochs"
            type="line"
            data={[
              { name: 'Epoch 0', value: 2.5 },
              { name: 'Epoch 10', value: 1.8 },
              { name: 'Epoch 20', value: 1.2 },
              { name: 'Epoch 30', value: 0.8 },
              { name: 'Epoch 40', value: 0.5 },
              { name: 'Epoch 50', value: 0.3 },
            ]}
            color="#10B981"
          />
        </div>
      )
    },
    {
      label: "Code",
      content: (
        <div>
          <CodePlayground 
            title="Gradient Descent Simulation"
            initialCode={`// Simple gradient descent simulation
let x = 5;  // Starting point
const learningRate = 0.1;
const iterations = 10;

// Objective: minimize f(x) = x^2
console.log('Minimizing f(x) = x^2');
console.log('Starting x:', x);

for (let i = 0; i < iterations; i++) {
  const gradient = 2 * x;  // df/dx = 2x
  x = x - learningRate * gradient;
  const loss = x * x;
  console.log(\`Iteration \${i + 1}: x = \${x.toFixed(4)}, loss = \${loss.toFixed(4)}\`);
}`}
          />
        </div>
      )
    }
  ]}
/>

## Tips for Using Interactive Components

1. **Code Playground**: Great for teaching programming concepts, algorithms, or demonstrating code behavior
2. **Interactive Charts**: Perfect for visualizing data trends, model performance, or statistical concepts
3. **Quizzes**: Excellent for reinforcing learning and testing comprehension
4. **Tabs**: Useful for presenting multiple perspectives or implementations of the same concept
5. **Audio Player**: Essential for audio processing tutorials or podcast-style content

## How to Use in Your MDX Files

To use these components in your MDX blog posts, simply include them as JSX elements:

```mdx
<CodePlayground 
  title="Your Title"
  initialCode={`// Your code here`}
/>

<InteractiveChart 
  title="Chart Title"
  type="line"
  data={yourDataArray}
/>

<Quiz 
  title="Quiz Title"
  questions={questionsArray}
/>

<Tabs 
  tabs={tabsArray}
/>

<AudioPlayer 
  src="path/to/audio.mp3"
  title="Audio Title"
/>
```
